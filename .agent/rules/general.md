# General Programming Instructions

## General Principles

- Follow functional programming principles where possible.
- Adhere to Clean Code principles where possible.
- Follow common coding conventions.
- Prefer declarative over imperative code.

## Function and Module Design

- Keep functions small.
- Ensure functions and modules have a single responsibility.
- Limit function parameters (no more than 3 if possible).
- Avoid deep nesting; prefer early returns.
- Prefer composition over inheritance.

## Code Style

- Use descriptive and meaningful names.
- Write self-documenting code, avoid unnecessary comments.
- Favor immutability (use const where possible).
- Avoid mutable state.
- Prefer pure functions over side effects.
- Keep files short; split large files into modules.
- Ensure consistent error handling.
